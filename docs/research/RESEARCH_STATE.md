# Recenter — Research State

## What is current

**No formal user research artifacts exist in this repository.** There is
no user interview data, no survey data, no usability testing report, no
persona documentation, and no market/competitive analysis committed
anywhere in this codebase.

The closest thing to recorded "research" is the **Design QA self-critique
process** referenced in `docs/company/DECISION_LOG.md` (D2) — an internal
audit (Usability / Behavioral / Visual / Accessibility findings, scored
against "Four Gates": Design, Behavioral, Product Philosophy,
Engineering Readiness) performed against the shipped app itself. This is
a design/engineering QA process, not user research, and should not be
cited as if it were.

## What has been decided

Nothing — no research-driven decision is recorded as such anywhere in
this repository. Product and design decisions recorded in
`docs/company/DECISION_LOG.md` were made via Board approval of product
specifications and design packages; none of those documents cite
underlying user research as their basis, at least not in anything
provided to or committed by engineering.

## What is deferred

Everything. No research function currently exists for this product as
far as this repository shows.

## What a future AI session should know

- **Do not invent or infer user research findings.** If a future prompt
  references "research says X" without a citable source, ask where that
  research lives before treating it as established fact.
- If real research material exists outside this repository (a separate
  research tool, interview recordings, a data platform), it has not been
  shared with or referenced by engineering to date. This file should be
  updated the moment that changes, with a real source, not a summary
  reconstructed from memory.
- The Design QA self-critique process (see `docs/design/DESIGN_STATE.md`
  and `docs/company/DECISION_LOG.md`) is the one recurring, structured
  evaluation process this project has — it is useful context, but it
  evaluates the *shipped product* against stated principles, not user
  needs directly.
